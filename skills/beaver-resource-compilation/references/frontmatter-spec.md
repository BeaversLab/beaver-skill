# Frontmatter Specification

## Schema

```yaml
---
title: 'Article Title Here'
tags:
  - tag1
  - tag2
  - tag3
summary: 'three or four sentences summary of the article content.'
author: 'Author Name'
created_at: '2026-03-15T10:30:00'
compiled_at: '2026-03-15T14:22:00'
source: 'https://example.com/article'
source_domain: 'example.com'
---
```

## Fields

| Field           | Required    | Type     | Description                                             |
| --------------- | ----------- | -------- | ------------------------------------------------------- |
| `title`         | Yes         | string   | Article title, generated by agent analysis              |
| `tags`          | Yes         | string[] | 3-8 topic tags, generated by agent analysis             |
| `summary`       | Yes         | string   | 1-2 sentence summary, generated by agent analysis       |
| `author`        | No          | string   | Author name from source frontmatter. Omitted if unknown |
| `created_at`    | Yes         | string   | Original creation datetime                              |
| `compiled_at`   | Yes         | string   | Compilation datetime (auto-generated)                   |
| `source`        | Yes         | string   | Original source URL or file path                        |
| `source_domain` | Conditional | string   | Top-level domain, only when `source` is a URL           |

## Field Resolution Rules

### `title`

Agent reads the file content and generates a concise, accurate title matching the content language.

### `tags`

Agent generates 3-8 relevant tags covering topic, technology, and domain.

### `summary`

Agent generates a 1-2 sentence summary of the core content.

### `author`

Priority:

1. Source markdown frontmatter `author` field
2. Omit if not found

### `created_at`

Priority:

1. Source markdown frontmatter: `date`, `created_at`, or `created` field
2. File system birthtime (`stat.birthtime`)

Format: `YYYY-MM-DDTHH:mm:ss`

### `compiled_at`

Current datetime when the compilation script runs.

Format: `YYYY-MM-DDTHH:mm:ss`

### `source`

Priority:

1. Source markdown frontmatter: `source`, `url`, or `link` field (preserve original value)
2. Source file absolute path

### `source_domain`

Only added when `source` is a URL (`http://` or `https://`).

Extraction: `new URL(source).hostname` with leading `www.` stripped.

Examples:

- `https://mp.weixin.qq.com/s/abc123` → `mp.weixin.qq.com`
- `https://www.example.com/article` → `example.com`
- `/Users/marco/inbox/note.md` → field omitted (not a URL)

## Filename Convention

Files are renamed based on the `title` field:

- Spaces → underscores (`_`)
- Special characters removed: `/ \ : * ? " < > |`
- Extension: `.md`
- Collision handling: append `_1`, `_2`, etc.

Examples:

- `"LLM Prompt Engineering"` → `LLM_Prompt_Engineering.md`
- `"React 服务端组件详解"` → `React_服务端组件详解.md`
- Duplicate → `LLM_Prompt_Engineering_1.md`
