# 🎨 小红书信息图系列生成器 (beaver-xhs-images)

简体中文

## 功能描述

**beaver-xhs-images** 是一个专为小红书（Xiaohongshu / RedNote）平台设计的 AI 信息图系列生成技能。它可以将任意长文内容（文章、笔记、评测等）自动拆解为 1–10 张风格统一、适合刷屏传播的卡通风格信息图，覆盖从封面到结尾的完整内容链路。

核心价值：**一键将长内容转化为小红书爆款图文系列**，省去排版设计、风格协调、文案拆解的繁琐工作。

🛠️ 版本说明：本项目基于 [技能 baoyu-xhs-images](https://github.com/JimLiu/baoyu-skills/tree/main/skills/baoyu-xhs-images) 进行二次开发。在保留核心逻辑的基础上，重点针对原版中的提示词模板硬编码模型引用和风格自动选择仅支持英文关键词的问题进行了修复，并新增了中英文双语内容信号匹配、标准化图片生成接口契约、图片质量验证与错误恢复机制，以及首次设置流程中的布局与语言偏好配置，以提升使用体验。

## 优点

- **10 种视觉风格 × 8 种信息布局**：风格与布局可自由组合（如 `notion + dense` = 知识卡片感的高密度排版），总计 80+ 种组合方式
- **智能内容分析**：自动识别内容类型（种草/干货/测评/教程/避坑等）、目标受众和爆款潜力，支持中英文双语内容信号匹配，据此推荐最优风格
- **3 套差异化大纲策略**：故事驱动型、信息密集型、视觉优先型，覆盖不同创作意图
- **全链路一致性**：通过参考图链（Reference Image Chain）+ 会话 ID 机制，确保系列图片在角色设计、配色、插画风格上保持统一
- **可定制偏好**：支持通过 EXTEND.md 保存个人偏好（水印、默认风格/布局、语言、自定义风格），首次设置引导覆盖水印、风格、布局、语言和存储位置，后续使用自动加载
- **图片质量验证与错误恢复**：每张图片生成后自动验证输出完整性与风格一致性；生成失败时自动重试，重试仍失败则跳过并在完成报告中标注，不阻塞整个流程
- **标准化图片生成接口**：通过接口契约定义图片生成技能的必需/推荐/可选能力，不绑定特定后端，可适配不同的 AI 图片生成工具
- **完整的修改工作流**：支持对已生成图片进行编辑、添加、删除，且自动管理文件编号和备份

## 使用场景

### 适合使用

- 将公众号文章 / 技术博客 / 产品评测转化为小红书图文帖
- 快速制作知识卡片、教程图解、对比图等信息图系列
- 需要统一风格的多图系列（如品牌推广、个人 IP 内容）
- 为已有内容快速生成适合中文社交平台的视觉呈现
- 批量制作不同策略的大纲方案以选择最佳方向

### 不适合使用

- 需要真实照片（非卡通/插画风格）的场景
- 单张独立海报设计（本技能专注于多图系列）
- 非信息图类型的设计需求（如 Logo、UI 设计）
- 需要精确到像素级控制的排版需求

## 使用方法

### 触发方式

通过以下关键词或命令激活：

```bash
# 指定文件路径
/beaver-xhs-images posts/ai-future/article.md

# 指定风格和布局
/beaver-xhs-images posts/article.md --style notion --layout dense

# 直接粘贴内容
/beaver-xhs-images
[粘贴你的文章内容]
```

也可通过自然语言触发，提及："小红书图片"、"小红书笔记"、"小红书配图"、"小红书种草"、"XHS images"、"XHS cards"、"RedNote infographics"等关键词。

### 基本流程

1. **加载偏好设置**：自动检查 EXTEND.md 配置文件，首次使用时会引导完成初始设置
2. **内容分析**：AI 深度分析你的内容（类型、受众、爆款潜力等），并向你确认理解是否正确
3. **生成 3 套大纲方案**：分别对应故事型、信息型、视觉型策略，每套配有推荐风格
4. **选择方案并确认风格**：选择一套大纲（或混合搭配），确认/调整视觉风格和元素
5. **逐张生成图片**：按确认的大纲和风格依次生成所有图片，每张完成后会报告进度
6. **输出完成报告**：汇总所有生成物，展示文件列表

## 参数说明

| 参数 | 是否必填 | 默认值 | 说明 |
|---|---|---|---|
| 内容源 | 是 | — | 文件路径或直接粘贴的文本内容 |
| `--style <name>` | 否 | 自动推荐 | 视觉风格：`cute` / `fresh` / `warm` / `bold` / `minimal` / `retro` / `pop` / `notion` / `chalkboard` / `study-notes` |
| `--layout <name>` | 否 | 自动推荐 | 信息布局：`sparse` / `balanced` / `dense` / `list` / `comparison` / `flow` / `mindmap` / `quadrant` |

## 依赖

| 依赖项 | 类型 | 是否必须 | 说明 |
|---|---|---|---|
| 图片生成能力 | 工具/技能 | 是 | 需要运行环境具备 AI 图片生成能力（如 DALL·E、Midjourney 等），具体接口要求见 `references/config/image-gen-interface.md` |
| 文件系统访问 | 运行时权限 | 是 | 需要读写文件的权限，用于保存分析、大纲、提示词和生成的图片 |
| Bash 命令执行 | 运行时权限 | 是 | 用于检查 EXTEND.md 配置文件是否存在 |

## 注意事项

- **两个确认点不可跳过**：Step 2（内容理解确认）和 Step 4（大纲 + 风格确认）是必须的交互环节，AI 不会擅自跳过
- **首次使用须完成初始设置**：如果未找到 EXTEND.md 配置文件，技能会阻塞在 Step 0，必须先完成偏好设置才能继续
- **敏感人物自动替换**：涉及真实人物肖像时，会自动用卡通替代方案处理
- **生成失败自动重试**：单张图片生成失败时会自动重试一次；重试仍失败则跳过该图片并继续生成后续图片，最终在完成报告中列出所有失败项
- **文件冲突自动处理**：同名目录/文件会追加时间戳，已有文件会自动备份
- **风格一致性依赖参考图链**：第 1 张图（封面）建立视觉锚点，后续所有图片引用第 1 张以保持风格统一

## 常见问题

**Q：如何选择最适合我内容的风格？**
**A：** 不指定 `--style` 时，技能会根据内容关键词自动推荐（如知识类内容推荐 `notion`，美妆类推荐 `cute`）。你也可以在 Step 4 确认环节中更换风格。

**Q：可以混合使用不同的大纲策略吗？**
**A：** 可以。在 Step 4 选择时选 "Combine"，指定从不同策略中取用的具体页面即可。

**Q：生成后想修改某张图片怎么办？**
**A：** 先修改对应的提示词文件（`prompts/NN-{type}-[slug].md`），然后重新生成该图片。技能会自动备份原始文件。

**Q：如何保存我的偏好避免每次重新设置？**
**A：** 首次使用时的初始设置会引导你配置水印、默认风格、默认布局、输出语言和存储位置，自动保存到 EXTEND.md。支持项目级（`.beaver-skills/beaver-xhs-images/EXTEND.md`）和用户级（`$HOME/.beaver-skills/beaver-xhs-images/EXTEND.md`）两种存储位置。

**Q：支持哪些语言的内容输入？**
**A：** 支持任何语言的源内容输入，AI 会自动检测。输出语言取决于你在首次设置中选择的语言偏好（支持中文、英文或自动检测）。

**Q：一次最多生成多少张图？**
**A：** 单次生成支持 1–10 张图片。封面占第 1 张，结尾占最后 1 张，中间为内容页。具体数量由内容量和策略类型决定。
